# Scaling to 10,000+ Users — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the 5 root causes that make Swellyo degrade at ~1k–10k users, so the app stays smooth at 10,000+ active users.

**Architecture:** Five **independent, separately-shippable workstreams**. Each produces working software on its own and can be reviewed/committed alone. Recommended order is by impact-to-effort (WS1 first), but they have no hard dependencies on each other.

## ⚠️ Project-wide guardrails (read before touching anything)

These come from the codebase memory and apply to **every** workstream:

- **Migrations are applied MANUALLY** via the Supabase SQL editor. **Never run `supabase db push`.** Remote migration history is frozen at `20260528`; migration files in the repo are *reference copies* that must be hand-applied to prod.
- **The notification push path is LIVE and UNGATED** — real users get real pushes. **Never invoke `dispatch-notification-queue` against prod to "test."** Use `NOTIFICATIONS_QUEUE_SHADOW='true'` and extracted unit tests only (WS4).
- **The group-trip broadcast trigger is LIVE on prod** and fires on 8 tables. Editing it is a hot-swap (WS2).
- **`match_surfers` SQL is built but NOT yet applied to prod**, and is uncommitted on `ohad` (WS5). Fold changes into that same un-applied file.
- **SECURITY DEFINER functions need explicit `GRANT ... TO authenticated`** or signed-in calls 403. Preserve every `REVOKE ... FROM public, anon` + `GRANT ... TO authenticated`.
- **Tests must never hit the network/DB** — the supabase client is mocked per-suite (`jest.config.js`).
- **Do not commit** unless asked — Ohad reviews and commits manually.
- **Verify realtime on a physical device, not the iOS Simulator** — the simulator fails the WebSocket upgrade and produces false connection errors (memory: `project_realtime_socket_and_scaling`).

### Task 1.4: Build the realtime load harness

**File:** Create `loadtest/realtime-footprint.js` (a standalone Node script, run manually — never imported by the app).

- [ ] **Step 1: Write the harness.** It must mimic the real footprint per virtual user: authenticate (seeded JWT) → open the socket → subscribe to `presence:user:{id}` (with `.track`) + `user-inbox:{id}` (private broadcast) → re-track every 60s → upsert `user_activity` every ~3 min.
```js
// loadtest/realtime-footprint.js
// Usage: SUPABASE_URL=... SUPABASE_ANON_KEY=... node loadtest/realtime-footprint.js 500
const { createClient } = require('@supabase/supabase-js');
const N = parseInt(process.argv[2] || '100', 10);
const clients = [];
async function spawn(i) {
  const sb = createClient(process.env.SUPABASE_URL, process.env.SUPABASE_ANON_KEY, {
    realtime: { heartbeatIntervalMs: 45000 },
  });
  // NOTE: private channels require a valid JWT (Realtime Authorization / RLS on realtime.messages).
  // Seed test users + sign in here, or the inbox/messages subscribes will CHANNEL_ERROR.
  const presence = sb.channel(`presence:user:test-${i}`, { config: { presence: { key: `test-${i}` } } });
  presence.subscribe((s) => { if (s === 'SUBSCRIBED') presence.track({ online_at: Date.now() }); });
  const inbox = sb.channel(`user-inbox:test-${i}`, { config: { private: true } });
  inbox.subscribe();
  clients.push({ sb, presence, inbox });
}
(async () => {
  for (let i = 0; i < N; i++) { await spawn(i); await new Promise(r => setTimeout(r, 20)); } // ramp
  console.log(`Holding ${N} connections. Watch the Realtime dashboard "concurrent peak".`);
})();
```

### Task 2.1: Add a per-user "my trips changed" topic + lighten the Explore signal

**Current:** the trigger sends to the single global `'trips-list'` topic on every `group_trips`/`group_trip_participants` change (`:60-73`). `trip:{id}` (per-trip, members) already exists (`:49-58`).

- [ ] **Step 1: Edit the trigger SQL.** In `broadcast_trip_change`, replace the topic-#2 block (`:60-73`) so that:
  - A **per-member** signal fires for participant changes: loop `SELECT user_id FROM group_trip_participants WHERE trip_id = v_trip_id` and `realtime.send(..., 'trips_mine_changed', 'trips-mine:' || user_id::text, true)` for each — **wrapped in the existing `exception when others` guard** so a broadcast failure never aborts the trip write.
  - The global `'trips-list'` topic fires **only** on `group_trips` INSERT or a status change (active/cancelled), **not** on every participant join/leave (participant-count churn is the firehose; Explore cards tolerate slightly stale counts).

```sql
-- replaces lines 60-73 in broadcast_trip_change()
if TG_TABLE_NAME = 'group_trips'
   and (TG_OP = 'INSERT'
        or (TG_OP = 'UPDATE' and NEW.status is distinct from OLD.status)
        or TG_OP = 'DELETE') then
  begin
    perform realtime.send(v_payload, 'trips_list_changed', 'trips-list', true);
  exception when others then null; end;
end if;

-- per-member "your trips changed" fan-out for participant/trip changes
if TG_TABLE_NAME in ('group_trips', 'group_trip_participants') then
  begin
    for v_member in
      select user_id from group_trip_participants where trip_id = v_trip_id
    loop
      perform realtime.send(
        v_payload, 'trips_mine_changed',
        'trips-mine:' || v_member.user_id::text, true);
    end loop;
  exception when others then null; end;
end if;
```

```ts
export function useTripsListRealtime(userId: string | undefined) {
  const queryClient = useQueryClient();
  // ...existing 300ms coalesce timer...
  useEffect(() => {
    const exploreCh = supabase
      .channel(TRIPS_LIST_TOPIC, { config: { private: true } })
      .on('broadcast', { event: 'trips_list_changed' },
          () => coalesce(() => queryClient.invalidateQueries({ queryKey: tripsKeys.explore })))
      .subscribe();
    if (!userId) return () => { supabase.removeChannel(exploreCh); };
    const mineCh = supabase
      .channel(tripsMineTopic(userId), { config: { private: true } })
      .on('broadcast', { event: 'trips_mine_changed' },
          () => coalesce(() => queryClient.invalidateQueries({ queryKey: ['trips', 'my', userId] })))
      .subscribe();
    return () => { supabase.removeChannel(exploreCh); supabase.removeChannel(mineCh); };
  }, [userId, queryClient]);
}
```
