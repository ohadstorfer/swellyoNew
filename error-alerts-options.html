How should we get alerted when a user hits an error?

Plain-English comparison so you can pick. Nothing is built yet.

👉 My recommendation

  • Use Sentry as the main tool (web + iOS + Android + Edge Functions).
  • Turn on PostHog error tracking too — it's free and already installed.
  • Don't build a custom system. It looks cheap but breaks fast (see below).
  • Alerts go to your phone (push) + email — your pick.
Total cost: $26/month · Setup: about half a day
The 4 options

Sentry RECOMMENDED

$26/mo (Team plan)
  • Catches web, mobile & native crashes
  • Readable error reports (real file + line)
  • Groups repeats — no spam
  • Push + email + Slack alerts
  • Costs money for good alerts

PostHog ADD-ON (FREE)

$0 — already installed
  • Free up to 100k errors/month
  • Shows what the user did before the error
  • ~2 lines of code to switch on
  • Misses true native crashes
  • Known mobile bug — backup only

Build it ourselves SKIP

$0 but costs time + headaches
  • Error reports are unreadable (minified)
  • One bug = 1000s of messages
  • Can't catch native crashes
  • Only ok for a quick "site is down" ping

Supabase Log Drains LATER

+$60/mo add-on
  • Sends DB / server errors into Sentry
  • Pricey + bills per event
  • Not needed now
  • Instrumenting functions directly is free
