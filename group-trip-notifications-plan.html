Group Trip Notifications

A simple, smart plan — keep members informed, pull dormant users back, never spam.

Scope: group_trips Push + In-app feed Goal: inform + re-engage

1 Principles

Five rules every notification must pass.

  • Push only if it's timely AND useful right now. Everything else lives in the feed.
  • Always land on an action. Every tap opens the exact screen to do the next thing.
  • One person, one action = one notification. Batch the rest into a digest.
  • Respect quiet hours. Hold non-urgent pushes from 9pm to 8am local.
  • Never repeat. If they already saw it in the feed, don't push it too.

2 The golden rule: push vs feed

Decide the channel before anything else.

  • Push when it needs a decision, is time-sensitive, or is about them directly.
    e.g. you're approved · trip starts tomorrow · someone asked to join your trip
  • Feed only when it's nice-to-know group activity that can wait.
    e.g. someone claimed gear · gear list was edited · a 4th member joined
Quick test: "If they never open the app, does this matter?"  Yes → push.  No → feed.

When NOT to send (anti-spam guardrails)

  • Don't re-engage anyone active in the last 24 hours.
  • Max 1 re-engagement push per user every 3 days.
  • Back off after 2 ignored re-engagement pushes in a row.
  • Never during quiet hours (9pm–8am local).
  • Never if the user muted that trip or turned off re-engagement.

5 Smart rules — the engine behind it

Cross-cutting logic that keeps the whole system calm and trustworthy.

  • Batch. Group rapid events — 3 joins in an hour become one "3 people joined".
  • Frequency cap. Max ~3 pushes per day per user across all their trips.
  • Quiet hours. Queue non-urgent pushes and release them at 8am local.
  • Dedup. If they read it in the feed, cancel the pending push.
  • Collapse. One trip = one live push — newest replaces the older one.
  • Mute per trip. Respect the existing mute; add a per-trip toggle.
  • Granular opt-out. Separate switches: reminders · group activity · re-engagement.
  • Priority order. Decisions > reminders > group activity > re-engagement. High priority can skip the batch.
