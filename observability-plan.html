Swellyo · Observability

Periodic checks + crash tracking — the plan

Scheduled tests that run on a timer + real-user crash reports, so we get alerted the second something breaks. Mobile-first · budget ~$0–30/mo. Nothing is built yet.

👉 My recommendation

  • Sentry (free) — catches real crashes & errors from real phones. Our #1 gap today.
  • DIY backend checks (free) — a scheduled function pings our critical Edge Functions + OpenAI + Google + S3, and shows health in our admin dashboard.
  • Better Stack (free) — an outside watchdog, in case our own backend (and our checker) is down.
  • Maestro (free, already installed) — run UI flows locally before each release. Cloud is $250/mo → skip.
Total cost: $0/month · Optional Sentry upgrade for Slack = $26/mo later
1 · Two kinds of monitoring (we need both)
TypeWhat it isCatchesMisses
Passive error tracking SDK in the app reports real crashes & errors from real users. Crashes, white screens, broken buttons Backend down at 3am with no users online
Synthetic checks A robot hits our key endpoints on a timer and checks the answer is right. Backend / OpenAI / Maps / S3 down, 24/7 Client-only UI bugs on a real screen
Plain "is it up?" pings are not enough — login can return "200 OK" and still be broken. We check the answer, not just the status code.

3 · Tools — Layer A: crash / error tracking

Sentry RECOMMENDED

Free (5k errors/mo) → $26/mo
  • Real crashes from real phones — our #1 gap
  • Readable reports (real file + line)
  • Auto source maps on EAS builds
  • After each OTA we must upload source maps

Firebase Crashlytics ALT

Free forever, unlimited
  • Truly free, no event cap
  • Great native crash symbolication
  • Misses JS-layer errors
  • Heavier setup, needs dev build

Bugsnag PRICEY

Free (7.5k) → $59/mo
  • Nice "stability score"
  • Big jump to paid tier

PostHog (we have it) NOT YET

Free 100k errors/mo
  • Already installed, generous free tier
  • RN crash capture is broken (bug, Mar 2026)
  • Revisit once the bug is fixed
3 · Tools — Layer B: backend synthetic checks

DIY (pg_cron + Edge Fn) RECOMMENDED

$0 · ~1–2 days build
  • Tests our real authenticated logic
  • Checks OpenAI / Google / S3 keys
  • Results live in our own dashboard
  • Dies if Supabase is down → need Layer C

Checkly BUY ALT

Free 10k runs/mo → $24/mo
  • Best hosted scripted checks
  • Half-day setup, multi-region
  • Alerts live in their dashboard, not ours
3 · Tools — Layer C: outside watchdog · Layer D: mobile UI

Better Stack WATCHDOG

Free (10 monitors, 30s)
  • Fastest free check interval
  • Email + Slack + free status page
  • Alerts us even if our backend is down
  • Simple pings only (that's fine here)

Maestro — local UI TESTS

$0 · already installed
  • Run flows before each release for free
  • Cloud scheduling = $250/mo → skip
  • Revisit cloud if we add CI + funding
Optional, later: Supabase exposes a free Prometheus metrics endpoint → pipe to Grafana Cloud (free) for DB/infra alerts. ~1–2h setup.

5 · Where alerts go

How you'll be notified

  • Email — Sentry, Better Stack & DIY checks all email on failure. Zero setup.
  • Admin / analytics dashboard — DIY checks write to a monitoring_runs table; a "System health" panel shows green/red per path + last failure.
  • Rule — only alert after 2 failures in a row for the same check, to avoid noise from one-off blips.
6 · Suggested rollout
  • Sentry (½ day) — install Expo SDK, ship in next build. Instant value: see real crashes.
  • Better Stack (1h) — add monitors on Supabase + key Edge Function URLs. Instant outside watchdog.
  • DIY checks (1–2 days) — health-monitor Edge Fn + pg_cron every 5 min + monitoring_runs table + email alerts (Resend).
  • Dashboard panel (½ day) — surface health inside the admin / analytics dashboard.
  • Maestro habit — run UI flows before each release. Optional later: Grafana infra metrics.
